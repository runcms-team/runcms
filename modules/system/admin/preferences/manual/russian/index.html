Основные настройки сайта

Генеральная конфигурация Вашего сайта.

E-Mail администратора:
Основной адрес электронной почты, на который отправляются все уведомления.

Почтовая функция:
По умолчанию, mail() - почта (там где это реализовано) отправляется посредством PHP. Некоторые хостеры не разрешают отправлять почту средствами PHP, в этом случае нужно выбрать опцию email() - любая почта с сайта будет отправлена только посредством вашего почтового клиента (Outlook Express и т.д.).

Язык по умолчанию:
Язык для вашего сайта. Если языковые файлы по каким либо причинам отсутствуют или недоступны, то по умолчанию будет использован английский язык.

Модуль стартовой страницы:
Список текущих установленных модулей. При выборе конкретного модуля стартовая страница сайта, будет загружать данный модуль.

Часовой пояс по умолчанию:
Если пользователь не настроит свой часовой пояс, то будет использована эта временная зона по умолчанию. Временная зона сервера определяется автоматически.

Скин по умолчанию:
Скин (графическая тема оформления) для Вашего сайта.

Позволить пользователям менять скины?
Если разрешено, то пользователи могут выбирать установленные скины в настройках своей учетной записи.

Ник для анонимных гостей:
Имя по умолчанию для анонимных (незарегистрированных) посетителей сайта.

Минимальная длина пароля:
Пароль меньшей длины не будет пропущен при новой регистрации.

Позволить анонимам оставлять комментарии?
Разрешить или нет анонимным (незарегистрированным) посетителям оставлять комментарии к новостям и т.д..

Максимум входящих сообщений для пользователя.
Число приватных сообщений, которое разрешено хранить на сайте. 0=неограниченно

Разрешить HTML-теги в комментариях?
Разрешить пользователям использовать HTML-теги в комментариях или нет.

Показывать внешние картинки в сообщениях, комментариях и подписях?
Данная опция запрещает или разрешает использовать тег <img src="">

Опции библиотеки изображений:
Мы можем разрешить или запретить использовать библиотеку изображений, разрешить или нет закачивать изображения на сайт, установить ширину и высоту закачиваемых картинок, а так же размер изображения в байтах (1 Кб = 1024 байта).

Разрешить регистрацию на сайте?
Включить/Выключить регистрацию новых пользователей

Включить защиту от спама? Требуется GD!
Данную защиту включают, чтобы предотвратить автоматическую регистрацию различных ботов и защитить сайт и его пользователей от спама. После включения этой опции при регистрации и авторизации нужно будет вводить специальный код безопасности.

Автоматически активировать новых пользователей при регистрации?
Если у Вас возникли проблемы с почтой или Вы хотите пропустить процесс активации новой учетной записи, включите данную опцию.

Уведомлять о новой регистрации?
Если включить данную опцию, то пользователям выбранной группы, будет отсылаться письмо при каждой новой регистрации.

Позволить пользователям удалять аккаунт?
Могут или нет пользователи самостоятельно удалять свою учетную запись.

Показывать картинку загрузки?
Картинка появляющаяся при загрузке страницы. Полезно при длительной загрузке страницы, как индикация того, что сайт продолжает загружаться.

Использовать gzip-сжатие?
Если Ваша версия PHP позволяет это, то включите данную опцию, это ускорит работу сайта.

Как строго отслеживать разрешенные символы в никах?
Возможны три варианта:

  1. Строго (только буквы и цифры)

  2. Средне

  3. Легко (рекомендуется для многобайтных символов)

Имя для cookies, в котором будет сохраняется ник пользователя в течение года:
Если это cookie существует, то ник будет автоматически подставлен в поле логина. Внимание!  Оставлять стандартное имя (exoops_user) - это снижение безопасности, измените его.

Имя сессии/cookies.
Дает пользователю возможность остаться в системе, пока время сессии/cookies не истекло. Представляет собой или имя для хранения ID сессии или имя переменной необходимой при авторизации с использованием только cookies. Внимание! Оставлять стандартное имя (exoops_session) - это снижение безопасности, измените его.

Время нахождения на сайте:
Это время (в сек.), по истечению которого пользователь будет автоматически исключен из системы и вынужден пройти авторизацию заново (1 час = 3600 сек).

Заметка: Авторизация основывается на IP пользователя, поэтому при смене IP, авторизацию придется повторить.

Использовать для авторизации сессии вместо cookies:
Если мы используем cookies, то данные необходимые для авторизации на сайте хранятся на стороне клиента, если сессии, то на стороне сервера, последнее более безопасно.

Активировать баннерную систему на сайте?
Включить или нет показ баннеров на сайте. Предварительно должен быть установлен модуль banners. Более подробно смотрите руководство к этому модулю.

Уровень отладки:
В данной опции мы устанавливаем варианты работы встроенного отладчика.

Ошибки: Включает  показ ошибок PHP. Полезно при отладке модулей. Это внутренний отладчик ошибок, который используется вместо стандартного отладчика ошибок PHP.

Время: Время генерации страницы (выводиться внизу страницы).

Инфо: Количество sql-запросов к базе данных, число загруженных файлов .php (размер) и размер html-строки, отсылаемой пользователю (выводиться внизу страницы).

Инфо и лог: В дополнение к предыдущему параметру показывает список (имя/путь) загруженных файлов .php  и список всех sql-запросов (эмуляция вывода команды EXPLAIN, для анализа связывания таблиц).

Визуальные данные: Показывает sql-запросы прямо на странице CMS, полезно для отладки модулей. Вы можете установить Xdebug для получения дополнительной информации о class/function.

Время кэширования страниц, мин:
Если время установлено, то все открываемые посетителями страницы будут кэшироваться - сохраняться в файлы (папка cache/system). При следующем открытии страницы, она не будет генерироваться заново, а загрузиться из файла кэша (пока не истечет время). Данная возможность предназначена в первую очередь для снижения нагрузки на базу данных, за счет уменьшения SQL запросов. Но мы все таки не рекомендуем ее включать, при большом количестве посетителей в единицу времени, так как нагрузка связанная с постоянными операциями над файлами, может оказаться больше.

Способ отображения комментариев по умолчанию:
Комментарии могут отображаться как в виде ленты (один комментарий следует за другим), так и в виде дерева (дерево заголовков комментариев). В данной опции мы устанавливаем способ отображения по умолчанию для всех посетителей сайта, если в их индивидуальных настройках не установлено другое.

Порядок отображения комментариев по умолчанию:
Порядок вывода комментариев (Новые первыми/Старые первыми) по умолчанию для всех посетителей сайта, если в их индивидуальных настройках не установлено другое.

Настройки аватара:
Мы можем разрешить или запретить загрузку изображения аватара, установить ширину и высоту закачиваемых картинок, а так же размер изображения в байтах (1 Кб = 1024 байта).

HTML-теги, которые администраторы и пользователи могут использовать на сайте:
Опция которая устанавливает, какие теги HTML могут использовать зарегистрированные пользователи и администраторы в сообщениях на сайте, а какие нет. Кроме того, запрещенные теги которые не прошли проверку при сохранении сообщения/новости и т.д. при показе этого сообщения/новости будут вырезаны.

Соответственно для администрации устанавливается один набор тегов HTML, для пользователей другой.
