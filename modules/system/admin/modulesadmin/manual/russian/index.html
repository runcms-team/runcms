Управление модулями

В данном разделе администрирования, можно осуществлять управление модулями сайта. Первая страница разделена на 2 части - установленные и не установленные модули.

Модуль
Имя модуля, логотип и ссылка на главную страницу администрирования этого модуля.

Версия
Текущая версия модуля.

Последнее обновление
Время и дата последнего обновления модуля, а так же его статус: "активный" (модуль установлен) и "не установлен".

Действие

Обновить:
Используется при обновлении (переустановки) модуля. Это полезно, когда Вы внесли изменения в модуль и хотите увидеть их в навигационном меню. Также это сбрасывает все настройки модуля. Соответственно и все настройки блоков этого модуля так же сбрасываются. Информация из базы данных при обновлении не удаляется.

Заметка: Обновление системного модуля (администрирование сайта) приведет к сбросу настроек ВСЕХ модулей. Полезно для полного сброса и обнуления данных всей системы.

Деактивировать:
Отключение текущего модуля без удаления.

Установка:
Установка модуля. (данная опция невидна, если модуль уже установлен)

Удаление:
Полное удаление модуля из системы, связанных с ним блоков, а так же всех таблиц этого модуля из базы данных!

Без изменений:
Оставить все как есть.

Порядок
Порядок расположения ссылок на модули в меню навигации (если этот модуль открыт для данной группы). Можно удалить модуль из меню, установив 0.

Заметка: Установка в 0 только скроет ссылку, без удаления самого модуля. подробнее смотрите руководства: Администрирование/Группы для настройки прав доступа и Администрирование/Блоки для настройки блоков.

Инфо
Небольшое окошко с различной полезной информацией о модуле: версия, автор, описание, лицензия и т.д..

Редактировать
Переход на страницу редактирования модуля. Кроме системного модуля, и модулей без заглавной страницы (только блок(и)).

Информация:

Информация о модуле в более сжатом виде (версия, статус, последнее обновление, установочный каталог).

Параметры:

Имя модуля, порядок, действие (обновить, удаление и т.д.)

Права доступа:

Устанавливаем права доступа, на просмотр информации и администрирование этого модуля разными группами.

Дополнительно:
Для переименования имени (названия) модуля в меню навигации, измените переменную $modversion['name'] в файле xoops_version.php, который должен находиться в корневой папке модуля. Если имя представляет из себя константу, например _MI_NEWS_NAME, то название нужно менять в языковом файле: /modules/(каталог модуля)/language/russian/modinfo.php

После внесения изменений в модуль, его нужно обновить.
